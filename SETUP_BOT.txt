================================================================================
   ANTIGRAVITY DISCORD BOT – SETUP-ANLEITUNG
   Schritt-für-Schritt: Bot auf der Discord-Website einrichten
================================================================================


═══════════════════════════════════════════════════════════════════════════════
 SCHRITT 1: Discord Developer Portal öffnen
═══════════════════════════════════════════════════════════════════════════════

1. Öffne deinen Browser und gehe zu:
   https://discord.com/developers/applications

2. Melde dich mit deinem Discord-Konto an.
   (Das Konto, mit dem du den Bot verwalten willst)


═══════════════════════════════════════════════════════════════════════════════
 SCHRITT 2: Neue Application erstellen
═══════════════════════════════════════════════════════════════════════════════

1. Klicke oben rechts auf den Button "New Application"

2. Gib einen Namen ein, z.B.: "Antigravity Bot"

3. Akzeptiere die Discord Terms of Service

4. Klicke auf "Create"

5. Du bist jetzt auf der "General Information"-Seite.
   → WICHTIG: Kopiere die "APPLICATION ID" (auch "Client ID" genannt)
   → Trage sie in deine .env Datei ein als:
     CLIENT_ID=hier_die_kopierte_id


═══════════════════════════════════════════════════════════════════════════════
 SCHRITT 3: Bot-User erstellen
═══════════════════════════════════════════════════════════════════════════════

1. Klicke links im Menü auf "Bot"

2. Unter "Build-A-Bot" siehst du deinen Bot

3. Klicke auf "Reset Token" (oder "Add Bot" falls noch keiner existiert)

4. Bestätige mit "Yes, do it!"

5. → WICHTIG: Kopiere den BOT TOKEN sofort!
   Er wird nur einmal angezeigt!
   → Trage ihn in deine .env Datei ein als:
     DISCORD_TOKEN=hier_der_kopierte_token

   ⚠️  ACHTUNG: Diesen Token NIEMALS teilen oder öffentlich posten!
   Jeder mit dem Token kann deinen Bot kontrollieren!


═══════════════════════════════════════════════════════════════════════════════
 SCHRITT 4: Bot-Einstellungen konfigurieren
═══════════════════════════════════════════════════════════════════════════════

Auf der "Bot"-Seite, scrolle runter und stelle folgendes ein:

1. "PUBLIC BOT"
   → AUS (deaktivieren) – damit nur du den Bot zu Servern einladen kannst

2. "PRESENCE INTENT"
   → AN (aktivieren)

3. "SERVER MEMBERS INTENT"
   → AN (aktivieren)

4. "MESSAGE CONTENT INTENT"
   → AN (aktivieren)

5. Klicke auf "Save Changes" unten


═══════════════════════════════════════════════════════════════════════════════
 SCHRITT 5: Bot zu deinem Discord-Server einladen
═══════════════════════════════════════════════════════════════════════════════

1. Klicke links im Menü auf "OAuth2"

2. Klicke dann auf "URL Generator"

3. Unter "SCOPES" setze folgende Häkchen:
   ☑ bot
   ☑ applications.commands

4. Unter "BOT PERMISSIONS" setze folgende Häkchen:

   Allgemein:
   ☑ Read Messages/View Channels

   Text-Berechtigungen:
   ☑ Send Messages
   ☑ Send Messages in Threads
   ☑ Embed Links
   ☑ Attach Files
   ☑ Read Message History
   ☑ Use Slash Commands

   (Optional, für vollständige Funktionalität:)
   ☑ Manage Messages

5. Scrolle nach unten – dort siehst du die "GENERATED URL"

6. Kopiere diese URL

7. Öffne die URL in deinem Browser

8. Wähle den Discord-Server aus, auf dem der Bot laufen soll

9. Klicke auf "Authorize"

10. Löse das Captcha

11. → Der Bot ist jetzt auf deinem Server! (erscheint als offline)


═══════════════════════════════════════════════════════════════════════════════
 SCHRITT 6: Deine Discord User-ID herausfinden (für ADMIN_IDS)
═══════════════════════════════════════════════════════════════════════════════

1. Öffne Discord (App oder Browser)

2. Gehe zu: Einstellungen → Erweitert → "Entwicklermodus"
   → AN (aktivieren)

3. Gehe zurück zum Chat

4. Klicke mit der RECHTEN Maustaste auf deinen eigenen Namen
   (in einer Nachricht oder in der Mitgliederliste)

5. Klicke auf "ID kopieren"

6. → Trage die ID in deine .env Datei ein als:
     ADMIN_IDS=hier_deine_user_id

   Für mehrere Admins, komma-getrennt:
     ADMIN_IDS=123456789,987654321


═══════════════════════════════════════════════════════════════════════════════
 SCHRITT 7: (Optional) Server-ID für schnelle Command-Registrierung
═══════════════════════════════════════════════════════════════════════════════

Wenn du Commands sofort testen willst (ohne 1 Stunde zu warten):

1. Aktiviere den Entwicklermodus (siehe Schritt 6.2)

2. Klicke mit der RECHTEN Maustaste auf deinen SERVER-NAMEN
   (ganz oben links über den Channels)

3. Klicke auf "ID kopieren"

4. → Trage die ID in deine .env Datei ein als:
     GUILD_ID=hier_die_server_id

   HINWEIS: Mit GUILD_ID werden Commands NUR auf diesem Server
   registriert. Ohne GUILD_ID werden sie global registriert
   (funktioniert überall, dauert aber bis zu 1 Stunde).


═══════════════════════════════════════════════════════════════════════════════
 SCHRITT 8: .env Datei fertigstellen
═══════════════════════════════════════════════════════════════════════════════

Deine .env Datei sollte jetzt so aussehen:

   DISCORD_TOKEN=MTIzNDU2Nzg5MDEyMzQ1Njc4.XXXXXX.XXXXXXXXXXXXXXXXXXXXXXXX
   CLIENT_ID=123456789012345678
   ADMIN_IDS=987654321098765432
   GUILD_ID=111222333444555666

(Die Werte oben sind nur Beispiele – benutze deine eigenen!)


═══════════════════════════════════════════════════════════════════════════════
 SCHRITT 9: Bot starten
═══════════════════════════════════════════════════════════════════════════════

Öffne ein Terminal im Projektordner und führe aus:

   # 1. Slash Commands registrieren (einmalig)
   npm run deploy

   # 2. Bot starten
   npm start

   # Oder für Entwicklung (Auto-Restart bei Änderungen):
   npm run dev

Wenn alles klappt, siehst du:
   ✅ Bot ready! Logged in as AntgravityBot#1234
   📡 Serving 1 server(s)

Der Bot erscheint jetzt als ONLINE auf deinem Discord-Server!


═══════════════════════════════════════════════════════════════════════════════
 SCHRITT 10: Erste Schritte als Admin
═══════════════════════════════════════════════════════════════════════════════

──── 10.1  Antigravity-Instanzen registrieren ────

   Bevor der Bot etwas tun kann, musst du mindestens eine
   Antigravity-Instanz registrieren.

   Beispiel – eine lokale Instanz auf Port 8765:
   /instance add name:mein-ag port:8765

   Beispiel – Instanz mit Beschreibung:
   /instance add name:mein-ag port:8765 description:Mein Haupt-Antigravity

   Beispiel – Instanz auf einem anderen Rechner:
   /instance add name:buero-ag port:9333 host:192.168.1.50

   Beispiel – mehrere Instanzen für verschiedene Projekte:
   /instance add name:projekt-alpha port:8765 description:Frontend-Projekt
   /instance add name:projekt-beta port:9223 description:Backend-Projekt
   /instance add name:test-ag port:9224 description:Zum Testen

   Alle registrierten Instanzen anzeigen:
   /instance list

   → Zeigt: 🟢 (verbunden) oder 🔴 (nicht verbunden) pro Instanz


──── 10.2  Instanz verbinden ────

   Nach dem Registrieren musst du die Verbindung herstellen.
   Antigravity muss dabei mit --remote-debugging-port laufen!

   Beispiel – Verbindung herstellen:
   /instance connect name:mein-ag

   Beispiel – Verbindung trennen:
   /instance disconnect name:mein-ag

   Beispiel – Instanz komplett entfernen:
   /instance remove name:test-ag

   Tipp: Der Bot verbindet sich auch automatisch, wenn ein User
   /ask an eine noch nicht verbundene Instanz schickt.


──── 10.3  User für Instanzen freischalten ────

   Standardmäßig haben User KEINEN Zugriff. Du musst jeden
   User einzeln für jede Instanz freischalten.

   Beispiel – User Max für eine Instanz freischalten:
   /permit grant user:@Max instance:mein-ag

   Beispiel – User Max für mehrere Instanzen freischalten:
   /permit grant user:@Max instance:projekt-alpha
   /permit grant user:@Max instance:projekt-beta

   Beispiel – Zugriff einer Instanz entziehen:
   /permit revoke user:@Max instance:projekt-beta

   Beispiel – ALLEN Zugriff eines Users entziehen:
   /permit revoke user:@Max
   (Ohne instance-Angabe werden ALLE Instanzen entzogen)

   Beispiel – Berechtigungen eines Users anzeigen:
   /permit list user:@Max
   → Zeigt alle Instanzen und Commands, die Max benutzen darf


──── 10.4  Commands für User freischalten ────

   Selbst mit Instanz-Zugriff kann ein User keine Commands
   benutzen, bis du sie einzeln oder alle auf einmal freigibst.

   Beispiel – ALLE Commands auf einmal freischalten (empfohlen):
   /command-access grant-all user:@Max

   Beispiel – nur bestimmte Commands freischalten:
   /command-access grant user:@Max command:ask
   /command-access grant user:@Max command:model
   /command-access grant user:@Max command:files
   /command-access grant user:@Max command:status

   Beispiel – einen Command wieder sperren:
   /command-access revoke user:@Max command:admin
   (So kann Max z.B. alles außer Admin-Commands nutzen)

   Beispiel – Command-Berechtigungen eines Users anzeigen:
   /command-access list user:@Max
   → Zeigt: ✅ /ask, ✅ /model, ❌ /admin, usw.


──── 10.5  Modelle verwalten ────

   User können ihr Modell selbst wählen, aber du kannst als
   Admin Modelle erzwingen oder blockieren.

   Beispiel – Modell für einen User erzwingen (Lock):
   /model lock user:@Max model:gemini-2.5-pro
   → Max kann NUR dieses Modell benutzen, egal was er einstellt

   Beispiel – Modell-Lock aufheben:
   /model unlock user:@Max
   → Max kann wieder frei wählen

   Beispiel – ein bestimmtes Modell für einen User blockieren:
   /model block user:@Max model:gpt-4o
   → Max kann alles außer GPT-4o benutzen

   Beispiel – Block wieder aufheben:
   /model unblock user:@Max model:gpt-4o

   Beispiel – Standard-Modell eines Users direkt setzen:
   /admin set-model user:@Max model:claude-sonnet-4-20250514


──── 10.6  Testen ────

   Jetzt kannst du den Bot testen!

   Beispiel – einfache Frage an die Standard-Instanz:
   /ask message:Hallo, wie geht es dir?

   Beispiel – Frage an eine bestimmte Instanz:
   /ask message:Erkläre mir React Hooks instance:projekt-alpha

   Beispiel – längere Anfrage:
   /ask message:Schreibe eine Python-Funktion die Fibonacci berechnet

   Beispiel – Status aller Instanzen prüfen:
   /status

   Beispiel – Admin-Dashboard anzeigen:
   /admin status

   Beispiel – .md Dateien anzeigen:
   /files list
   /files view path:README.md

   Beispiel – Hilfe anzeigen (funktioniert für alle):
   /help


──── 10.7  User komplett zurücksetzen ────

   Falls du alle Einstellungen und Berechtigungen eines Users
   auf einmal löschen willst:

   Beispiel:
   /admin reset-user user:@Max
   → Löscht: Instanz-Zugriffe, Command-Berechtigungen,
     Modell-Einstellungen, Modell-Sperren

 FEHLERBEHEBUNG
═══════════════════════════════════════════════════════════════════════════════

Problem: "DISCORD_TOKEN not set"
→ Stelle sicher, dass die .env Datei im Hauptverzeichnis liegt
→ Prüfe ob der Token korrekt kopiert wurde (keine Leerzeichen)

Problem: "Missing Access" oder "Missing Permissions"
→ Bot hat nicht genug Rechte auf dem Server
→ Lade den Bot erneut ein mit den korrekten Permissions (Schritt 5)

Problem: Commands werden nicht angezeigt
→ Führe "npm run deploy" aus
→ Mit GUILD_ID: sofort sichtbar
→ Ohne GUILD_ID: bis zu 1 Stunde warten

Problem: Bot ist offline
→ Stelle sicher, dass "npm start" läuft
→ Prüfe die Konsole auf Fehlermeldungen

Problem: "Invalid Token"
→ Gehe zum Developer Portal → Bot → Reset Token
→ Kopiere den neuen Token in die .env Datei
